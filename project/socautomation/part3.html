SOC Automation Project Part 3

Overview

This is part 3 of the SOC Automation Project. This part demonstrates how to deploy an agent in Wazuh. This part of the project also showcase how to generate telemetry, ingest sysmon logs into Wazuh. Lastly, demonstrates how to create an index pattern and rule in Wazuh.

Project Goal

- Implement a SOC environment with Wazuh as the SIEM/XDR, TheHive for case management and integrate it to Shuffle for SOAR capabilities.

Tools and Technologies

- Wazuh
- Sysmon
- Mimikatz
- Windows 10

Wazuh Agent Deployment, Generate Telemetry, Index and Rule Creation

Objectives

-Deploy Wazuh agent in windows 10 host.
-Create an index pattern.
-Create a custom rule.
-Ingest sysmon logs into Wazuh.
-Generate telemetry using Mimikatz.

What I did

1. Deployed wazuh agent in Windows 10 host using powershell
2. Configured Wazuh to ingest sysmon logs
3. Configured Wazuh to archive logs and created an index pattern for it. 4. Generated telemetry using Mimikatz in Windows 10 host. 5. Created a rule for the generated telemetry to test Wazuh capabilities.

Skills Demonstrated

- Incident Detection and Response
- SIEM Configuration and Rule Creation
- Endpoint Security and Monitoring
- Log Management
- Security Testing
- Threat Detection

Key Takeaway

This part of the project deepened my understanding on how to configure the SIEM, create rules for detection/response, and monitor endpoint in the SIEM application.
