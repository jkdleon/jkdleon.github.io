SOC Automation Project Part 4

Overview

This is part 4 of the SOC Automation Project. This part demonstrates how automation can help ease the tasks of an analyst in a SOC environment. In this part of the project Shuffle was used the SOAR to automate the alert creation in TheHive for alert/case management.

Project Goal

- Implement a SOC environment with Wazuh as the SIEM/XDR, TheHive for case management and integrate it to Shuffle for SOAR capabilities.

Tools and Technologies

- Wazuh
- Sysmon
- Mimikatz
- Windows 10
- TheHive
- Shuffle

Automation

Objectives

- Automate the alert creation in TheHive with data from Wazuh.
- Use VirusTotal for to analyze the suspicious file(Mimikatz).

What I did

1. Signed up for a free account in Shuffle
2. Signed up for a free account in VirusTotal
3. Created a new organization and user in TheHive instance.
4. Generated telemetry in Windows 10 host using Mimikatz.
5. Automated the alert creation in TheHive.

Skills Demonstrated

- Automation

Key Takeaway

This part of the project deepened my understanding on how SOAR works and how it helps in easing the workload of an analyst in their day to day tasks. It also let me understand that even if the tool already has the input fields given to you, you will still run in an error and that leads in enhancing troubleshooting skills. This part of the project also shows that a tool is just a tool and we can use different tool to have our work get better. Skills like troubleshooting is still the best foundation a person should have.
