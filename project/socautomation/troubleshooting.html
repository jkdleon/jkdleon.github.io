SOC Automation Project - Troubleshooting

Overview

This page is all about the problems I encountered and how I troubleshoot it.

Problems

1. Error in installing cassandra service.
2. Error in configuring cassandra and elasticsearch service.
3. Error in integrating TheHive and Shuffle.
4. Problem in the created json body when using the fields given by TheHive app in Shuffle.

Error in installing cassandra service

Problem Encountered

When installing cassandra service I've run into an error stating that the package is not available eventhough I've already importedt the GPG key and added Cassandra's repository.

Troubleshooting

What I did to resolve this issue is to directly download the package from cassandra website into TheHive machine.
After downloading the package directly into the machine, installation did not encounter errors anymore.

Error in configuring cassandra and elasticsearch service.

Problem Encountered

When configuring both cassandra and elasticsearch, I've run into an IP address binding error. Upon checking the public address given by Azure was not binding in the service. I used the public address given by the cloud provider in this project but best practice to use private addresses in configuring cassandra and elesticsearch.

Troubleshooting

What I did to resolve this issue is to host the TheHive instance in DigitalOcean since is not allowing to bind their public address to services. Upon hosting again the instance in DigitalOcean, I have not run into any errors configuring the services.

Error in integrating Wazuh and Shuffle.

Problem Encountered

In making the workflow in Shuffle and configuring Wazuh, I run into an error: unable to enable integration for: 'shuffler'. File not found inside 'integrations'.

Troubleshooting

I checked my ossec.conf file again if I just mistyped the name of the integration but the configuration is correct. After that I looked where 'shuffler' is mentioned and found that it was only mentioned in the hook_url line in the integration block. I've tried using the backup ossec.conf but I still run into the error. I also tried to use a link shortener to check if 'shuffler' in the hook_url was just being misinterpreted but still run into the error. I searched the web and found that cleaning up the old queue files might help in resolving the issue. I cleaned up the old queue files, restarted the Wazuh manager and tested again. Now I did not run into any errors.

Problem in the created json body when using the fields given by TheHive app in Shuffle.

Problem Encountered

The problem I encountered was that the variable was not being replaced by the value I inputted in the given field by TheHive app.

Troubleshooting

What I did is to check the given error and tried to go from there. One by one I tried to fix the error but when I tried to fix 1 error, another different error would come up. Checked the apps resolved issues if I can find any relevant document about it but found nothing. In TheHive app, there is an advance option where you can create your own json for the app to be used, so instead of forking the app and editing the fields inside, I tried using the advance option and created the json body, editing all the necassary line that had an error and run the workflow again. After using the advance option, no errors encountered and an alert was created in TheHive dashboard.

Key Takeaway

1. Always have a fallback plan like manual installation when package repositories fail.
2. Persistent errors may be due to residual files.
3. Cloud provider limitations.
4. If the UI form fails, use the manual option if available.
5. Taught me how to stay persistent and use alternative approach in resolving problems.
